Hexo + github搭建blog

Posted on 2019-04-08 | In Hexo |

安裝Hexo

STEP:

  1. 先安裝node.js download , 因為hexo 是基於node.js所建立的靜態網頁

  2. 用npm -v , 來檢查是否安裝成功

  3. 開一個資料夾 , 用來放hexo ,並且在此資料夾開啟CMD

  4. 使用npm指令來安裝hexo

    npm install -g hexo-cli

  5. 安裝完成後 , 初始化blog(任意取名)

    hexo init blog

  6. 執行hexo指令 , 確認是否安裝完成

    hexo g

    hexo d

    hexo s

    hexo s , 預設網址為localhost:4000

和github做連結

STEP :

1.打開git bash , 輸入:

git config –global user.name “你的GitHub使用者名稱”

git config –global user.email “你的GitHub信箱”

2.生成SSH密鑰

ssh-keygen -t rsa -C “你的GitHub註冊信箱”

然後連續按三個enter就行了

然後找到生成的.ssh的文件夹中的id_rsa.pub，將内容全部複製，打開 setting_key 然後 new SSH key , 將剛剛複製的內容全部貼上

4.輸入

ssh git@github.com

看到這樣就成功了，這裡之所以設置GitHub密鑰原因是，通過非對稱加密的公鑰和私鑰来完成加密，公鑰放置在GitHub上，私鑰放置在自己的电脑里。GitHub要求每次推送code都是合法用戶，所以每次推送都需要輸入帳號密碼驗證推送用戶是否是合法用戶，為了節省輸入帳號密碼的步驟，採用了ssh，當每次推送時，git就會匹配你的私鑰跟GitHub上面的公鑰是否是配對的，若匹配則為合法用戶，就可以推送code。

5.修改 _config.yml (此為最外層的) , 拉到最下面修改成

deploy:
type: git
repository: ssh://git@github.com/你的名子/你的名子.github.io.git
branch: master

(原本repository , 開頭為https , 但我碰到了一個錯誤為 [用戶名]被[用戶名]禁止存取,所以改成上述那樣)

6.最後打開CMD到放hexo的資料夾輸入

npm install hexo-deployer-git –save

hexo clean

hexo g

hexo d

hexo s

即完成hexo + github 建立blog！
